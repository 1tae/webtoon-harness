# codex-image 동시성 규약 (계약 §5)

> **이 문서는 codex-image를 사용하는 모든 에이전트(panel-artist-a/b/c, ref-sheet-artist, prompt-smith 등)가 공유하는 동시성 규칙이다.**

## 핵심 규칙

- codex 전역 동시 세션은 **최대 5개**다. ChatGPT 플랜의 동시 요청 한도 때문에 6개+는 큐잉으로 응답이 들쭉날쭉해지고 일부 작업이 비정상적으로 길어진다.
- 따라서 **한 번에 최대 5장만 배치 렌더**한다. 자기 분량이 17장이면 5+5+5+2의 웨이브로 처리한다.
- panel-artist-a/b/c가 동시에 떠 있어도 **codex exec 동시 실행 총합은 5를 넘지 않는다.** 오케스트레이터가 아티스트 렌더 패스를 순차 디스패치하므로, **자기 차례(디스패치 신호)에만** 5장 배치를 인플라이트로 돌린다.
- 신호 없이 선제 렌더로 한도를 깨지 않는다.

## 적용 에이전트별 참고

| 에이전트 | 적용 방식 |
|---------|----------|
| **panel-artist-a/b/c** | scene 그룹별 5장 배치 렌더. 오케스트레이터의 디스패치 신호에만 실행. |
| **ref-sheet-artist** | 캐릭터 레퍼런스 시트도 5장 배치 한도를 지킨다. 패널 렌더와 동시에 codex를 점유하지 않는다(레퍼런스는 패널 렌더 전에 끝낸다). |
| **prompt-smith** | 패널 프롬프트 설계 시 panel-artist들이 5장씩 배치 렌더할 수 있도록 패널을 정렬·그룹화한다. |

## 안티패턴

- **6장 이상 한 배치** — 큐잉으로 응답 분산 증가, 일부 작업 비정상 지연.
- **여러 아티스트가 동시에 각자 5장씩(총 10장+)** — 전역 5 한도 초과. 반드시 순차 디스패치.
- **포그라운드 직렬 실행** — 병렬 효과 상실. 항상 `run_in_background: true`.
- **`sleep`로 완료 대기** — 백그라운드 통지가 자동. 폴링 금지.
- **동일 출력 경로 N개 동시 사용** — 마지막만 남음. 파일명 전부 달라야 함.

## 한도 초과 대응

- codex 세션 한도 초과 신호(큐잉/지연): 배치 크기를 5 이하로 줄이고 다음 디스패치 신호를 기다린다.
- 응답이 450초+ 걸리면 플랜 한도 의심 → `codex login status`로 확인, 동시 수를 줄여 재시도한다.
