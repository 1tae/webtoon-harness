# in-image 텍스트 베이크 규칙

> **이 문서는 in-image 텍스트 베이크에 관여하는 모든 에이전트(prompt-smith, letterer, panel-validator, quality-reviewer, episode-compositor, art-director)가 공유하는 규칙이다.**

## 핵심 원칙: 후작업/오버레이 절대 금지

이 하네스의 **모든 텍스트(말풍선 대사·효과음·화면 UI·환경 문자)**는 codex 이미지 생성 시 **작화에 함께 그려진다.** HTML 오버레이도, 포토샵 타이핑도, 어떤 후작업 합성도 없다.

## 프롬프트 규칙

- **"no text" 부정 프롬프트를 쓰지 않는다.** 대신 부정 프롬프트는 `no watermark, no English text, no gibberish/garbled text, no misspelled text`로 둔다(텍스트 자체는 허용하되 깨진/영어/오탈자만 배제).
- 한글 텍스트는 **원문 그대로 따옴표로** 명시한다.
- 위치는 lettering.md의 좌표/방향대로, **얼굴·핵심 작화를 가리지 않게**.
- 무대사 패널은 말풍선 지시를 넣지 않는다.

## 통합 레터링 (integrated lettering)

베이크의 목적은 텍스트가 작화의 일부로 보이는 것이다:
- 말풍선과 한글은 **작화와 같은 손그림 잉크 톤**으로 그려져 장면에 녹아들어야 한다.
- **깨끗한 디지털 고딕 폰트를 평평하게 위에 얹은 "붙여넣기" 느낌은 실패** — 독자가 "텍스트를 후작업으로 얹었다"로 오해한다(EP01 P30·P33 피드백).
- 프롬프트에 `hand-lettered into the artwork in the same bold black comic ink as the linework, integrated with the drawing (NOT a flat typeset/digital-font label pasted on top)`를 포함한다.

## 오버레이-룩(OVERLAY) 판별 신호

하나라도 있으면 C3(d) REGEN (철자가 완벽해도):
- 획 두께가 기계적으로 균일, 작화 잉크선의 강약·번짐·끊김과 따로 논다
- 자간·행간·베이스라인이 자로 잰 듯 완벽 균등(손글씨 불규칙성 0)
- 시스템 고딕(나눔고딕·맑은고딕·Noto Sans 류) 디지털 산세리프 룩
- 글자 가장자리가 작화보다 과하게 매끈/선명, 그레인·질감이 그림과 불일치
- 텍스트가 "레이어 위에 떠 있는" 느낌 — 검정 톤·채도·해상도가 그림과 분리
- 말풍선 윤곽은 손그림인데 글자만 디지털(또는 반대)로 재질이 어긋남

**INTEGRATED(ACCEPT) 신호:** 획 두께가 라인워크 잉크와 같은 계열 + 약간의 유기적 불규칙성, 같은 그레인·검정 톤, 같은 펜으로 그린 듯 작화에 녹아듦.

## 말풍선 종류별 시각 규약

| 종류 | 시각 스타일 |
|------|-----------|
| **대사(dialogue)** | 흰 배경·검은 테두리·둥근 모서리, 꼬리(tail)가 화자를 가리킨다 |
| **생각(thought)** | 점선 테두리·기울임, 꼬리 없음(또는 구름 점) |
| **외침(shout)** | 두꺼운 테두리·굵은 글씨·약간 기울기, 뾰족 폭발형 |
| **나레이션(narration)** | 사각 박스, 꼬리 없음. **남발 금지** — 대사로 전할 수 있으면 대사로 |
| **계약자(collector)** | 잉크블랙 풍선+꼬리 없음+금빛 글자 (작품 고유) |

## 한글 베이크 주의사항 (최대 리스크)

diffusion 이미지 생성은 한글을 자주 깨뜨린다:
- 말풍선당 대사를 **짧게 끊는다**(어절 수 적게)
- 대본 원문 철자를 **정확히** 적는다
- panel-validator의 C3(텍스트 정확/가독) 재생성 루프로 통과시킨다
- 긴 대사는 여러 풍선/패널로 분할한다
- 3회 실패 시 "말풍선 모양 유지 + 가장 정확한 텍스트 버전 채택"으로 마감

## 적용 에이전트별 역할

| 에이전트 | 역할 |
|---------|------|
| **letterer** | 말풍선 베이크 명세(`ep{NN}_lettering.md`) 설계 |
| **prompt-smith** | 베이크 명세를 codex 프롬프트에 담아 이미지에 함께 생성 |
| **art-director** | 말풍선 시각 규약(종류별 스타일)을 style-bible에 규정 |
| **panel-validator** | C3 축에서 한글 정확도/가독/통합 레터링 검증, REGEN 판정 |
| **quality-reviewer** | 끝단 2차 검수에서 오버레이-룩 독립 교차 비교 |
| **episode-compositor** | 말풍선이 이미 이미지에 있으므로 **텍스트 오버레이를 얹지 않는다** |

## 레퍼런스 시트 예외

캐릭터 레퍼런스 시트(refs/)만은 **텍스트 없이** 렌더한다:
- 프롬프트에 `no text, no labels, no speech bubbles, no watermark`
- 깨끗한 외형 도감이어야 기준이 된다 — 패널의 in-image 말풍선과 반대
